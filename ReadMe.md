# Word Frequency Counter with SHA-256

Написать программу, которая считает частоту слов в текстовом файле, 
но учитывает только те слова, для которых SHA-256 хэш строки 
(предыдущее слово + текущее слово) заканчивается на символ от 0 до 7 
в шестнадцатеричном представлении.

Условия:

1. Первое слово в тексте:
   * Пропустить, так как для него нет предыдущего слова.
2. Подсчёт частоты:
   * Если условие выполняется, увеличить счётчик соответствующего слова.
3. Формат результата:
   * Вывести в файл список слов с их частотой, отсортированный по убыванию частоты.

Требования:
1. Многопоточность:
   * Распараллелить обработку текста: текст делится на блоки, 
     и каждый блок обрабатывается в отдельном потоке.
2. SHA-256:
   * Использовать простую реализацию SHA-256, которая легко интегрируется в проект.
3. Выходные данные:
   * Сохранить результаты в файл в формате: <слово>: <частота>.

Ограничения (необходимо уточнить):
 - Размер файла? Допустим, что помещается в память.
 - Может ли слово содержать: цифры, ', _, -. Предполагаем, что да.
 - Слова могут быть в верхнем и нижнем регистре? Да.
 - Слова отличающиеся регистром считаем разными? Да.
 - количество потоков. Предполагаем: мин = 1, макс = 64.
 - SHA-256 - использована реализаци SHA под публичной лицензией
             https://create.stephan-brumme.com/hash-library/
             https://github.com/stbrumme/hash-library/blob/master/readme.md

## Использование

```bash
word_frequency.exe <input_file> <output_file> [number_of_threads]

# Пример:
word_frequency.exe input.txt output.txt 4
```

**Параметры:**
- `input_file` - входной текстовый файл
- `output_file` - выходной файл с результатами
- `number_of_threads` - количество потоков (1-64, опционально)

## Результаты

Программа создаст:
- `output.txt` - список слов с частотой
- `word_frequency.log` - логи выполнения

## Реализация

### Архитектура программы

Программа использует многопоточную обработку текста с локальной агрегацией результатов для минимизации синхронизации 
между потоками.

#### Основные компоненты:

1. **Чтение файла** (`readFileToWords`)
   - Построчное чтение входного файла
   - Токенизация по whitespace
   - Удаление пунктуации в конце слов
   - Сохранение в `std::vector<std::string>`

2. **Разделение на блоки** (`splitWordsForThreadsBalanced`)
   - Равномерное распределение слов между потоками
   - Перекрытие границ блоков для корректной обработки пар слов
   - Динамическое уменьшение количества потоков при недостаточном количестве слов

3. **Обработка текста** (`processBlock`)
   - Каждый поток обрабатывает свой блок слов независимо
   - Для каждого слова (начиная со второго) вычисляется SHA-256 хэш пары (предыдущее_слово + текущее_слово)
     Используется Public portable C++ Hashing Library (https://github.com/stbrumme/hash-library/blob/master/readme.md)
   - Если хэш заканчивается на символ 0-7, слово добавляется в локальный счётчик
   - Локальная `std::unordered_map<std::string, size_t>` используется для минимизации конкуренции за мьютекс
   - После обработки локальные результаты мёржатся в глобальную unordered_map под защитой мьютекса

4. **Сортировка результатов**
   - Копирование из `unordered_map` в `std::vector<std::pair>`
   - Сортировка по убыванию частоты, затем по алфавиту для одинаковых частот

5. **Запись результатов** (`writeResultsToFile`)
   - Вывод отсортированных результатов в формате `слово: частота`
